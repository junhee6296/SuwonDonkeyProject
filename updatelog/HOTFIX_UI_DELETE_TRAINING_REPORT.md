# Hotfix: UI / 삭제 복구 / 학습 명령 생성

## 1. 삭제 방식 복구
- 삭제 표시는 단순 상태 플래그가 아니라 실제 이미지 파일을 `data/_deleted_backup` 폴더로 이동하도록 변경했습니다.
- 프레임 목록에서는 삭제된 항목을 계속 남겨 빨간색으로 표시합니다.
- `삭제 이미지 복구`를 누르면 `_deleted_backup`의 이미지를 원래 위치로 되돌립니다.
- 삭제/복구 상태는 `data/_ui_marks.json`에 저장되며, 프로그램 재실행 후에도 유지됩니다.

## 2. 프레임 목록 색상 표시
- 삭제된 프레임: 빨간색 계열 배경.
- 교체/편집된 프레임: 초록색 계열 배경.
- 선택된 항목도 삭제/편집 상태 색상이 우선 표시되도록 처리했습니다.

## 3. 다중 선택 UI 문구 및 비율
- `체크 해제` 문구를 `전체 해제`로 변경했습니다.
- `전체 선택`과 `전체 해제` 버튼이 같은 가로 길이를 갖도록 레이아웃을 수정했습니다.

## 4. 삭제/복구 UI 분리 및 필터 UI 정리
- 삭제/복구 버튼을 재생 바 아래 행에서 분리하고, `이미지 삭제 / 복구` 그룹으로 별도 배치했습니다.
- 재생 행은 `이전 / 자동 재생 / 다음 / 저장`만 남도록 정리했습니다.
- `삭제 이미지만`, `교체/편집만` 필터가 다른 체크박스/숫자 입력과 겹치지 않도록 세로 배치로 정리했습니다.

## 5. 학습 명령 실행 개선
- 최초 실행 시 `학습 명령 생성`을 먼저 누르도록 안내합니다.
- `학습 명령` 텍스트 박스는 기본적으로 읽기 전용입니다.
- `수동 편집` 체크박스를 켜야 직접 수정할 수 있습니다.
- 환경 진단 버튼은 표시하지 않도록 유지했습니다.
- VirtualBox SSH 명령에 `BatchMode`, `NumberOfPasswordPrompts=0`, `ConnectionAttempts=1`, `ConnectTimeout`, `ServerAlive` 옵션을 추가하여 비밀번호 프롬프트/연결 대기 때문에 UI가 끝없이 기다리는 상황을 줄였습니다.

## 검증 메모
- 이 실행 환경에는 Visual Studio/dotnet CLI가 없어 실제 빌드 실행은 하지 못했습니다.
- 중괄호 균형, 주요 이벤트 핸들러, 레이아웃 메서드, 삭제/복구/학습 명령 생성 연결은 정적으로 점검했습니다.
