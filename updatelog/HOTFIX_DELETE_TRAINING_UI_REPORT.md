# Hotfix: 삭제/복구, 프레임 색상, 학습 명령 생성 UI

## 반영 내용

1. 이미지 삭제 방식 변경
- 삭제 버튼은 더 이상 목록에서 프레임을 제거하지 않습니다.
- 선택/체크된 프레임의 이미지 파일을 `data/_deleted_backup` 폴더로 이동합니다.
- 목록 항목은 유지되며 삭제된 프레임은 빨간 배경으로 표시됩니다.
- `삭제 이미지 복구` 버튼으로 `_deleted_backup`의 파일을 원래 위치로 되돌립니다.
- 삭제/복구 상태는 `data/_ui_marks.json`에 저장되어 재실행 후에도 유지됩니다.

2. 프레임 목록 표시 개선
- 체크 목록의 항목을 OwnerDraw 방식으로 직접 그려 삭제 프레임은 빨강, 교체/편집 프레임은 초록으로 표시합니다.
- `전체 선택`, `전체 해제` 버튼의 가로 폭을 동일하게 맞췄습니다.
- 삭제/편집 필터는 한 줄에 겹치지 않도록 별도 행으로 재배치했습니다.

3. 삭제/복구 UI 분리
- 재생 바 아래 버튼 줄에서 삭제/복구를 분리했습니다.
- `이미지 삭제 / 복구` 그룹을 새로 만들고 그 안에 `이미지 삭제`, `삭제 이미지 복구` 버튼을 배치했습니다.
- 재생 버튼 줄은 `이전`, `자동 재생`, `다음`, `저장`만 남도록 정리했습니다.

4. 학습 명령 생성 절차 강화
- 최초 실행 시 학습 명령 입력칸은 읽기 전용이며, 안내 문구만 표시됩니다.
- `학습 실행`을 바로 누르면 먼저 `학습 명령 생성` 창을 열도록 안내합니다.
- `수동 편집` 체크박스를 켜야만 명령어를 직접 수정할 수 있습니다.
- `환경 진단` 버튼은 UI에서 제거/숨김 처리했습니다.

5. VirtualBox/SSH 실행 안정화
- VirtualBox 명령 생성 시 SSH 옵션에 `BatchMode`, `NumberOfPasswordPrompts=0`, `ConnectionAttempts`, `ConnectTimeout`, `ServerAliveInterval`, `ServerAliveCountMax`를 포함했습니다.
- SSH가 비밀번호 입력을 기다리며 멈추는 상황을 줄였습니다.
- `&&`가 SSH/WSL 따옴표 내부에 있을 때 Windows cmd로 잘못 넘기지 않도록 shell 판정 로직을 보완했습니다.
