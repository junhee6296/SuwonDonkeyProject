# DonkeyCar UI 데이터 관리 도구

## 이번 수정 핵심

- Visual Studio WinForms 디자이너가 흰 화면으로 뜨지 않도록 UI 컨트롤을 `Form1.Designer.cs`에 실제 배치했습니다.
- `Form1.cs` 생성자에서 동적 UI를 만드는 구조를 제거하고, 디자이너에서도 동일한 버튼/레이아웃을 볼 수 있게 했습니다.
- 비밀 기능 1: 조향각 시계열의 이동평균과 변동성 밴드를 이용한 이상 주행 자동 탐지를 추가했습니다. 데이터 로드 시 자동 분석되고, 수동으로 다시 실행할 수도 있습니다.
- 비밀 기능 2: 이미지 위에서 마우스로 영역을 드래그한 뒤, 해당 영역을 가리거나 다른 이미지로 교체할 수 있게 했습니다.
- 편집 전 원본 이미지는 `data/_edited_backup`에 보관됩니다.
- 삭제된 이미지는 `data/_deleted_backup`에 보관되며, 직전 삭제 1회는 `삭제 취소`로 복구할 수 있습니다.


## Gemini 리뷰 반영 버그픽스

- 프레임 목록 텍스트 잘림을 줄이기 위해 목록 표시 형식을 짧게 바꾸고 가로 스크롤 폭을 자동 계산합니다.
- 이상 주행 탐지에서 이전 프레임 탐색이 O(N^2)로 커지던 부분을 O(N) 방식으로 개선했습니다.
- 이미지 마스킹/교체는 표시 중인 Bitmap을 직접 수정하지 않고 복제본에 적용합니다.
- catalog와 이미지 저장은 임시 파일 작성 후 교체하는 방식으로 안정성을 높였습니다.
- 학습 프로세스 시작 실패 시 핸들이 누수되지 않도록 `Process.Dispose()` 처리를 보강했습니다.

## 사용 방법

1. `TeamApp.sln`을 Visual Studio 2022로 엽니다.
2. `Form1.cs [디자인]`을 열면 실제 UI 컨트롤이 보입니다.
3. 실행 후 `폴더 열기`에서 DonkeyCar `mycar`, `data`, 또는 catalog/images가 있는 tub 폴더를 선택합니다.
4. 데이터 로드 직후 이상 주행 후보가 자동 표시됩니다. 필요하면 `이상 탐지 실행`을 눌러 다시 분석합니다.
5. `다음 이상` 버튼으로 빨간색 후보 프레임을 순차 탐색하고, `이상 주행 후보만 표시` 필터로 후보만 볼 수 있습니다.
6. 이미지 편집은 프레임 이미지를 마우스로 드래그해 영역을 선택한 뒤 `영역 가리기` 또는 `이미지로 교체`를 누르면 즉시 파일에 저장됩니다.

## 빌드 대상

- TargetFramework: `net8.0-windows`
- Windows Forms 사용: `UseWindowsForms=true`
